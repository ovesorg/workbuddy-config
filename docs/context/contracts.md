# Contracts

## What Goes In This Repo

This repo contains **shared, credential-free configuration artifacts** that the whole team uses:

- Connector MCP templates (JSON configs with env-var placeholders)
- Shared user skills (SKILL.md packages)
- Bootstrap scripts (bash / PowerShell)
- Documentation (agent manifests, reference, ADR)

## What Never Goes In This Repo

The following are **never committed** — they are local to each machine:

| Artifact | Reason |
|---|---|
| Real API tokens, keys, passwords | Security |
| `workbuddy.db` | Per-user local state |
| `user-state.json` | Per-user local state |
| `workspace-state.json` | Per-user local state |
| Session / trace data | Ephemeral |
| `*.key`, `*_secret*.json`, `*_config.json` | Credential patterns |
| `site/`, `build/` output | Build artifacts |

See `.gitignore` for the full exclusion list.

## Skill Contract

All skills in `skills/` must include:

- `SKILL.md` — the primary skill definition
- `agent_created: true` in frontmatter if created by an AI agent

Skills installed from the marketplace live in `~/.workbuddy/skills-marketplace/` and are **not** tracked in this repo (they are auto-regenerated by WorkBuddy).

## Connector Config Contract

Each connector in `connectors/` must include:

- `mcp.json` — the MCP server configuration
- No embedded credentials; use `${ENV_VAR}` placeholders

## Bootstrap Contract

Bootstrap scripts must:

- Use symlinks (Linux/macOS) or NTFS junctions (Windows)
- Never overwrite existing files; skip items that already exist
- Target `~/.workbuddy/` as the destination

## ADR Contract

Architecture decisions about this repo live in `docs/adr/`. Examples of decisions that belong here:

- Choosing symlinks vs. copies for bootstrap
- Which connectors to include or exclude
- Skill curation policy
- Breaking change handling

ADR files follow the naming pattern `####-title.md` (date + short slug).
